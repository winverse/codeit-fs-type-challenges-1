# type-challenge

이 디렉터리는 TypeScript 이론 절과 연결된 작은 타입 실습 모음입니다. 각 절 폴더에는 `README.md`, `template.ts`, `test-cases.ts` 세 파일만 두고, 해설은 루트 `solution` 폴더 아래에 따로 모아 둡니다. `README.md`는 실습 목표와 해야 할 일을 먼저 설명하고, `template.ts`에는 기본 문제와 추가 문제가 함께 들어 있으며, `test-cases.ts`는 학생 풀이가 의도한 타입을 만족하는지 확인합니다.

| 절 | 경로 | 다루는 핵심 |
| --- | --- | --- |
| 2-03 | `type-challenge/2-03.타입을-정하는-법` | 변수 타입 선언과 추론 결과를 구분하고, 같은 기준을 강의 데이터에 다시 적용합니다. |
| 2-05 | `type-challenge/2-05.기본형` | 기본형 다섯 가지를 실제 값에 다시 매핑하고, 같은 분류 기준을 판매 데이터에 적용합니다. |
| 2-07 | `type-challenge/2-07.배열과-튜플` | 길이가 자유로운 목록과 순서가 고정된 구조를 나누고, 세 칸 튜플까지 같은 기준으로 확장합니다. |
| 2-09 | `type-challenge/2-09.객체-타입` | 객체 구조, 선택 속성, 키-값 맵을 한 번에 정리하고, 같은 규칙을 회원 프로필 데이터에 반복 적용합니다. |
| 2-12 | `type-challenge/2-12.함수에-타입-정하기` | 함수 시그니처의 입력과 출력을 다시 읽고, 같은 규칙을 다른 로그 함수와 라벨 함수에 적용합니다. |
| 3-01 | `type-challenge/3-01.Enum` | Enum 멤버와 실제 값의 관계를 다시 확인하고, 방향값에도 같은 목록 방식을 적용합니다. |
| 4-01 | `type-challenge/4-01.Interface` | 공통 구조와 확장 구조를 다시 분리하고, 같은 분리 방식을 계정 도메인에 적용합니다. |
| 5-01 | `type-challenge/5-01.리터럴-타입` | 리터럴 추론과 widening을 다시 비교하고, 문자열 외의 리터럴에서도 같은 차이를 확인합니다. |
| 5-04 | `type-challenge/5-04.Union-타입` | 객체 Union과 리터럴 Union을 함께 다시 정리하고, 혜택 도메인에도 같은 구분 기준을 반복합니다. |
| 5-06 | `type-challenge/5-06.Intersection-타입` | 공통 속성 조각과 역할 조합을 다시 합치고, 같은 조합 방식을 전투 캐릭터에도 적용합니다. |
| 5-10 | `type-challenge/5-10.keyof와-typeof-연산자` | 값에서 타입을 가져오고 다시 키를 뽑는 흐름을 반복하고, 필터 설정 객체로 같은 관계를 다시 확인합니다. |
| 6-01 | `type-challenge/6-01.제네릭` | 제네릭 함수와 인터페이스를 함께 다시 연습하고, 다른 박스 구조와 선택 함수로 같은 감각을 반복합니다. |

실습은 각 절 폴더의 `README.md`를 먼저 읽고, `template.ts`를 수정한 뒤 `test-cases.ts`로 타입 검사 결과를 확인하면 됩니다. 해설은 `type-challenge/solution/{절 경로}/solution.ts` 아래에 따로 모아 두었으므로, 풀이를 끝낸 뒤 마지막에만 열어 비교하는 편이 적절합니다.

```bash
pnpm --package=typescript dlx tsc --noEmit type-challenge/2-03.타입을-정하는-법/test-cases.ts
```
